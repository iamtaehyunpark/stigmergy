# Cheonma Canonical Terminology Standard

## 1. Overview
This document defines the canonical terminology, casing rules, and error-code vocabulary for the Cheonma payments platform. All new development, documentation, and API contracts must adhere strictly to this standard. Legacy terms are deprecated and must not appear in new code or public documentation.

## 2. Casing and Naming Conventions

### 2.1 Endpoint Paths
- **Format**: `/{service}/{resource}`
- **Casing**: `kebab-case` for all path segments.
- **Pluralization**: Resources must be pluralized (e.g., `/transactions`, not `/transaction`).
- **Versioning**: Always prefix with `/v1/` (e.g., `/v1/checkout/transactions`).

### 2.2 Identifiers and Variables
- **Format**: `PascalCase` for types/classes, `camelCase` for variables/properties.
- **Prefixes**: No service-specific prefixes (e.g., use `TransactionId`, not `LedgerTxnId`).

### 2.3 Error Codes
- **Format**: `CHEONMA_{CATEGORY}_{CODE}`
- **Casing**: `UPPER_SNAKE_CASE`.
- **Categories**: `CORE`, `CHECKOUT`, `LEDGER`, `MERCHANT`.

# Cheonma Integration Tutorial (English)

This guide provides runnable code samples and migration notes for integrating with the Cheonma payments platform. All identifiers, endpoints, and error codes adhere to the [Cheonma Canonical Terminology Standard](root.1/canonical_standard).

## 1. Checkout Flow

To process a payment, initiate a transaction via the checkout service.

### Step 1: Create Transaction

**Endpoint:** `POST /v1/checkout/transactions`

```json
{
  "amount": 1000,
  "currency": "KRW",
  "merchantId": "MERCH_12345",
  "storeId": "STORE_001",
  "terminalId": "TERM_001"
}
```

**Response:**
```json
{
  "transactionId": "TXN_987654321",
  "status": "PENDING"
}
```

### Step 2: Confirm Transaction

**Endpoint:** `POST /v1/checkout/transactions/{transactionId}/confirm`

```json
{
  "transactionId": "TXN_987654321"
}
```

**Response:**
```json
{
  "transactionId": "TXN_987654321",
  "status": "COMPLETED"
}
```

## 2. Refund Flow

To issue a refund, use the refunds endpoint.

### Step 1: Create Refund

```json

  "amount": 500,
  "currency": "KRW",
  "reason": "Customer request"
}
```

**Response:**
```json
{
  "refundId": "REF_123456789",
  "status": "PROCESSING"
}
```

```json

}
```
