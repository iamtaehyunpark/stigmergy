# Rubric L2 — Vector database comparative report (frozen v1, before any E1 run)

Audience: an engineering team choosing a vector database.

Required elements (completeness):
1. Exactly three named vector database products.
2. For EACH product: features, pricing, and performance are all addressed.
3. Comparison across the products (not three isolated blurbs): comparable dimensions, trade-offs surfaced.
4. A final recommendation with an explicit justification tied to the comparison.

Accuracy: product characterizations plausible and internally coherent; no self-contradicting claims (e.g., "open-source" and "proprietary-only" for the same product).
Structure: parallel treatment of the three products; recommendation follows from the body.
Consistency: same product names/terms throughout; pricing/performance claims consistent between sections.
