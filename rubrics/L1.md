# Rubric L1 — Vector embeddings explainer (frozen v1, before any E1 run)

Audience: junior developers with no ML background.

Required elements (completeness):
1. Plain-language definition of a vector embedding (text -> numeric vector capturing meaning).
2. High-level mechanism of similarity search (nearness in vector space; e.g., cosine/nearest-neighbor at concept level).
3. Exactly the practical guidance asked: three do/don't tips grounded in application use (e.g., chunking, when NOT to use embeddings, model/dimension choice, caching).
4. One-page scope: concise; not a survey.

Accuracy: no incorrect mechanism claims (e.g., embeddings are not keyword indexes; similarity is not exact match).
Structure: readable top-down flow (what -> how -> tips) for the stated audience.
Consistency: terms used the same way throughout.
