# Rubric L3 — Field guide to LLM agent memory systems (frozen v1, before any E1 run)

Audience: engineers building LLM agents.

Required elements (completeness):
1. Four distinct chapters, each with substantive technical content (not stubs or placeholders).
2. Evidence of the per-chapter pipeline the task demands: researched content (specifics, not generalities) and edited, consistent prose per chapter.
3. A unified glossary covering terms actually used across the chapters.
4. A final assembled guide: the deliverable reads as ONE integrated document (chapter ordering, cross-references or at least uniform framing), not a loose concatenation.

Accuracy: technically sound treatment of agent memory topics (e.g., context windows, retrieval/vector stores, long-term memory architectures, multi-agent/shared memory) with no incorrect mechanism claims.
Structure: coherent book-level organization; chapters non-overlapping and complementary.
Consistency: terminology uniform across chapters AND matching the glossary; no chapter contradicting another.
